#Overview

ExamForge is a full-featured Single Page Application (SPA) for academic assessment. Supports course-based, topic-based, daily quiz, and mock exam modes. Uses Firebase Auth and Turso (edge SQLite) with a cache-first SyncManager layer.

Core Features
  • Adaptive quizzing (exam/practice modes)
  • EXA Rating system (ELO-based)
  • Streak tracking and analytics
  • Mock exams with anti-cheat
  • Daily quizzes with attempt tracking
  • Course library with topic hierarchy
  • Real-time inbox and schedule
  • Admin master panel (full CRUD)
  • PDF result sheets
  • Firestore-to-Turso migration
Tech Stack
  • Vanilla JS SPA
  • Turso (edge SQLite)
  • Firebase Auth
  • IndexedDB caching
  • Custom SyncManager
  • Express.js server
  • Cloudflare Workers
  • PWA enabled

#Data Flow

Read Path
UI Module -> SyncManager
  1. In-Memory Cache (instant)
  2. IndexedDB (LocalCache)
  3. Turso HTTP API
Write Path
UI Module -> DB Module -> DB Client
  -> Turso API -> Clear caches
Real-time
SyncManager.subscribe(path, cb, 30s)
  -> Poll -> Compare -> Notify

#DB Client Layer

src/db/client.js - Core HTTP client for Turso v2/pipeline API.

  • exec(sql, params?) Executes SELECT queries, returns array of row objects.
  • execOne(sql, params?) Same as exec but returns first row or null.
  • execute(sql, params?) Executes writes (INSERT/UPDATE/DELETE). Returns {affectedRows, lastInsertId}.
  • batch(statements) Executes multiple statements sequentially.
  • trackRead(label) No-op read tracking. Always returns false.

Window aliases: window.__execTurso = exec, window.__executeTurso = execute. Proxy URL: https://examforge-q88x.onrender.com/v2/pipeline

#SyncManager

sync.js (934 lines) - Cache-first Turso access layer with IndexedDB persistence, request deduplication, and polling subscriptions.

  • doc(path) Cache-first document fetch (e.g., users/abc123).
  • collection(path) Cache-first collection fetch.
  • query(path, constraints) Query with WHERE/ORDER BY/LIMIT.
  • collectionGroup(id, constraints) Collection group (SELECT * FROM table).
  • subscribe(path, cb, pollMs) Polling subscription on data change.
  • liveCollection(path, onChange, pollMs) Polling live collection.
  • unsubscribe(path, cb?) Remove subscriber and stop poller.
  • refresh(path, constraints?) Force refresh bypassing cache.
  • preload(paths) Pre-load documents at session start.
  • destroy() Clean up pollers and subscribers.

#LocalCache (IndexedDB)

cache.js (338 lines) - Zero-dependency IndexedDB wrapper. DB: examforge-cache, version 1, store: cache.

  • get(path) Retrieve entry by path. Returns {path, data, type, fetchedAt}.
  • set(path, data, type) Store entry (doc/collection/query).
  • delete(path), clear() Remove single or all entries.
  • getAllByType(type) Get entries filtered by type.
  • getStale(maxAgeMs), deleteStale(maxAgeMs) Age-based cache invalidation.
  • has(path), entries() Check existence and list all entries.

#Cloudflare Worker
